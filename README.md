# 舒写 Shuxie

面向创作者的智能写作与连载管理工作台。

## 项目愿景

打造一款面向创作者的智能写作与连载管理工具，让用户可以随时随地记录灵感、系统化生成作品大纲，并由“阅遍千万畅销书的 AI 写作导师”进行专业诊断与指导，最终高效完成并持续运营网文创作。

当前阶段先服务单一创作者的真实工作流，优先搭建可视化工作流与创作闭环，不追求完整产品化包装。

## 核心能力

- 灵感采集：接收碎片文字、语音转写、手写/OCR 内容
- 大纲生成：将零散想法整理为世界观、人物关系、主线剧情、节奏设计
- AI 写作：支持“人写 + AI 润色”与“AI 创写 + 人 Review”双模式
- 导师诊断：从畅销书与爆款网文规律出发，给出结构、节奏、市场适配和风险建议
- 连载管理：管理章节状态、存稿数量、发布节奏与平台运营建议

## MVP 范围

第一版聚焦三个闭环：

1. 灵感 -> 结构化创作素材
2. 素材 -> 大纲/章节
3. 定稿 -> 连载计划

暂不优先实现：

- 复杂权限系统
- 多人协作
- 完整自动发布接入
- 成熟商业化系统

## 推荐信息架构

- Dashboard：首页总控台
- Project Hub：作品库
- Idea Inbox：灵感采集台
- Outline Builder：大纲工厂
- Draft Studio：写作台
- Mentor Review：AI 导师诊断台
- Serial Planner：连载管理页

## 初始里程碑

### M1：产品骨架
- 明确页面结构
- 明确数据模型
- 明确工作流输入输出
- 搭建前端与服务目录骨架

### M2：创作核心链路
- 灵感录入
- 自动结构化
- 大纲生成
- 章节生成/润色
- AI 导师诊断

### M3：连载管理链路
- 存稿管理
- 排期管理
- 节奏建议
- 多平台扩展预留

## 开发原则

- 先做自己真的会用的工具
- 先做闭环，不做表演型功能
- 人工可控优先于全自动
- 结构化资产优先于单次对话
