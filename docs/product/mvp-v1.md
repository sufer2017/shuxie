# 《舒写》MVP 产品方案 v1

## 1. 产品定义

### 一句话定义
一套面向创作者的 AI 写作与连载管理工作台，帮助用户把零散灵感快速转化为可执行大纲、可迭代章节和可运营连载计划。

### 当前定位
- 当前只服务单一创作者（黄浩龙）
- 优先做可视化工作流
- 先跑通真实创作闭环
- 不追求完整商业产品包装

## 2. MVP 核心目标

### 目标 A：碎片灵感 -> 结构化创作资产
输出：
- 灵感摘要
- 类型标签
- 人物设定
- 世界观片段
- 情节冲突
- 爽点候选
- 可扩写方向

### 目标 B：创作资产 -> 可写大纲
输出：
- 作品定位
- 核心卖点
- 一句话梗概
- 角色关系
- 主线剧情
- 分卷思路
- 节奏建议
- 爽点分布

### 目标 C：大纲 -> 章节与连载节奏
支持：
- 章节草稿生成
- 章节润色
- AI 诊断
- 存稿管理
- 连载排期

## 3. MVP 暂不覆盖
- 复杂移动端 App
- 多人协作
- 成熟权限系统
- 复杂商业化体系
- 多平台自动发布实接
- 完整运营分析后台

## 4. 主要模块

### 4.1 Project Hub
作品项目管理、状态查看、待办汇总。

### 4.2 Idea Inbox
接收文本、OCR 文本、语音转写结果，沉淀为结构化灵感卡。

### 4.3 Outline Builder
把灵感池整理为世界观、人物、主线、分卷、节奏和爽点分布。

### 4.4 Draft Studio
支持“人写 + AI 润色”和“AI 创写 + 人 Review”。

### 4.5 Mentor Review
用 AI 导师视角做结构诊断、市场诊断、风险诊断与优化建议。

### 4.6 Serial Planner
管理章节状态、存稿数量、更新频率、发布队列和平台备注。

## 5. 页面建议
- Dashboard
- 作品详情页
- 灵感收集页
- 写作台
- AI 导师诊断页
- 连载管理页

## 6. 成功标准
第一版满足以下条件即可视为成功：

1. 可以持续录入灵感并自动归档
2. 可以从灵感池生成可用大纲
3. 可以围绕章节进行生成、润色、诊断
4. 可以看到可执行的连载计划
5. 整体流程可视化、可复用、可扩展
