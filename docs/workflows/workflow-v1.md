# 《舒写》工作流协议 v1

## 工作流 1：灵感录入 -> 结构化卡片
输入：
- 文本 / OCR 文本 / 语音转写文本

处理：
1. 识别所属作品
2. 提炼摘要
3. 判断灵感类型
4. 打标签
5. 产出扩写建议

输出：
- Idea 卡片
- 推荐去向（人物/世界观/剧情/章节）

## 工作流 2：灵感池 -> 大纲初稿
输入：
- 多条 Idea
- 题材/平台偏好

处理：
1. 聚合同类素材
2. 提炼卖点
3. 生成作品定位
4. 生成世界观/人物/主线/分卷/节奏
5. 导师诊断

输出：
- Outline
- Review（面向大纲）

## 工作流 3：大纲 -> 章节拆分
输入：
- Outline
- 指定分卷或剧情节点

处理：
1. 拆分章节目标
2. 生成关键冲突
3. 生成钩子与断章点
4. 生成章节待办列表

输出：
- Chapter 列表（todo）

## 工作流 4：章节草稿 -> 润色 / 诊断
输入：
- 章节草稿
- 风格要求

处理：
1. AI 润色
2. 节奏与情绪诊断
3. 爽点增强建议
4. 修改建议汇总

输出：
- polishedText
- Review（面向章节）

## 工作流 5：定稿 -> 连载计划
输入：
- 已定稿章节
- 平台信息
- 更新频率目标

处理：
1. 统计存稿数
2. 推荐排期
3. 生成未来 7 天发布建议
4. 标记 queued 状态

输出：
- SerialPlan
- 发布队列
